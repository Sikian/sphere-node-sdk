    Class BaseService Abstract Private

    Defined in: src/coffee/services/base.coffee

    Overview

    This class is abstract.
    This class is private.

    Define a BaseService to provide basic methods to interact with the HTTP API.

    This class should not be exposed and must be extended when defining a new *Service.

    Examples:

    class FooService extends BaseService
  @baseResourceEndpoint: '/foo'

    Promise ::process(fn, options = {}) Source

    Process the resources for each page separately using the function fn. The function fn will then be called once per page and has to return a Promise that should be resolved when all elements of the page are processed.

    Batch processing allows to process a lot of resources in chunks. Using this approach you can balance between memory usage and parallelism.

    Examples:

    # Define your custom function, which returns a promise
fn = (payload) ->
  new Promise (resolve, reject) ->
    # do something with the payload
    if # something unexpected happens
      reject 'BAD'
    else # good case
      resolve 'OK'
service = client.products
service.perPage(20).process(fn)
.then (result) ->
  # here we get the total result, which is just an array of all pages accumulated
  # eg: ['OK', 'OK', 'OK'] if you have 41 to 60 products - the function fn is called three times
.catch (error) ->
  # eg: 'BAD'

    Parameters:

    • (Function) fn — Function The function called for each processing page (it must return a Promise)
    • (Object) options — Object To configure the processing
      • (Boolean) accumulate — Boolean Whether the results should be accumulated or not (default true)

    Throws:

    • (Error) — Throws an Error if fn is not a Function

    Returns:

    • (Promise) — a Promise, fulfilled with an Array of the results of each resolved page from the `fn`, or rejected with an instance of an HttpError or SphereError
